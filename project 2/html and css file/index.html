Lesson 1: The Basics of the Web & HTML

World Wide Web
  • A collection of HTML documents made up of HyperText Markup Language
  • Everything found online (docs., images, videos) is glued together by HTML
  • Hyper Links connect all the HTML documents together making the World Wide Web, web-like

I open my favorite browser and and type in www.Udacity.com and hit go. My browser sends a request via the Internet using a protocol called HTTP to the servers hosting Udacity's files. The servers respond with the requested files, which my browser then displays. For more information, check out this video.

HTML

HTML is a language. It has its own words (tags) and rules for how those words can be put togehter. HTML is made up of 4 important components:

  1. Text Content (The actual text you see.)
  2. Markup (How the text looks, color, arangment, etc.)
  3. References to Other Documents (Images, Videos, Games etc.)
  4. Links to Other Pages (Hence, the web-like structure of the Web.)
HTML Markup

HTML Markup is made up of tags. These tags tell the computer how to display the text. A tag usually consists of both an opening and a closing tag, with the contents in between, and the entire entity is referred to as an element. Some tags, like the image tag, do not contain any text content to be displayed and therefore do not have an end tag. A tag can have none, one, or multiple attributes inside of it. This Video gives a good overview.

Inline v.s. Block Elements

HTML elements can be either inline, or block. Block elements form an invisible box around the contents inside of them. This box can be adjusted in both height and width. Inline elements simply appear in line with the text.

Lesson 2: Creating a Structured Document with HTML

The "Tree-Like" Structure of HTML

The DOM (Document Object Model) refers to the tree-like structure of a webpage. This tree-like, or branching structure is formed because HTML can have one tag nested inside another tag nested inside another tag and so on. When viewed with developer tools in a browser, this structure can be seen by expanding the triangle at the beginning of the first element (the html tag) and continuing to expand each tag until there is nothing left but text. What this triangle really represents though, is a box. You can think of a well structured webpage kind of like one of those pretty nesting dolls; but instead of the containers being doll shaped, the containers are box shaped. There is one big doll, or box (the html tag) and it contains all the other boxes, or elements, neatly boxed away inside of it.

Boxes

Everything on the web is made up of boxes. Even circles are boxes; they have just been styled, using CSS, to look like circles. This is becasue a browser sees a webpage as a series of boxes, nested inside of one another. When reading an HTML document as text, you can see a wave of changing indentations running along the left side. These indentations indicate how deeply the element's corresponding "box" is nested inside of the entire structure. This structure not only organizes all the elements within a page but it also makes the code easier to read. It also allows developers to easily re-arrange elements on the page.

Lesson 3: Adding CSS Style to HTML Structure

CSS

CSS stands for Cascading Style Sheets. Like HTML, it is also a language and it controls the look, or style, of a webpage. One HTML file can reference just one, or several CSS files in order to create the final, visible look, or style, of the page. You can think of CSS as a set of rules that tell the browser how to display certain elements. CSS allows programmers to group similar elements together by giving them the same class name and then specifying a style that should apply to all the elements with that same class name.

Specificity

CSS rules can be very general or very specific. Think of it in terms of laws, there are national laws, state laws, municipality laws and maybe even your own house rules. In CSS you can write rules that apply to the entire page, or just to one section, or just one box, or just one element. However, in CSS, the most specific rule is applied to every element, hence the word Cascading.The browser goes deeper and deeper into the set of rules to find the one that best describes the elements, and then applies that rule. This idea is known as inheritance.

Inheritance

Inheritance is a feature of CSS that helps developers avoid repetition. It relies on the tree-like, or nesting-box structure of HTML to operate. This structure creates an ancestor-descendant relationship in the elements and is what allows properties to be applied not only to a specified element, but also to its descendants. This means that descendant elements may inherit properties from any ancestor elements enclosing them. Inheritance is what prevents developers from having to declare certain properties over and over again.

Avoiding Repitition

Avoiding repetition when writing code is important for several reasons.

One, it helps prevent errors. When programmers don’t avoid repetition it leads to having to repeat the same, or just slightly different, code over and over again in many HTML elements. This leads to errors when a programmer needs to make changes to something. If they don’t meticulously make the same changes everywhere the repeated code appears, problems will arise.

Two, it provides consistency. Take this webpage for example. Each lesson looks the same. By giving each lesson div the same class, I am able to specify a style for that class in only one place. Then if I need to make changes, I can go back into the style rule for that class and easily revise it any way I want; I do not have to search the entire document for each instance of that code.

Lesson 1: Introduction to "Serious" Progarmming

Computers

Computers are machines that can be programmed to do almost anyting we want, as long as we can write a program that specifies a specific sequence of instructions.

Computer Program

A computer program is a specific sequence of steps a computer can follow to perform a particular function. Web browsers, mobile apps, games and even simple print statements are all examples of computer programs.

Programming Languages

A programming language is what programmers use to comunicate instructions to a computer. Python is one example of a programming language.

Python

Python is a programming language. The python interpreter is a computer program that interprets the code written in the python programming language and converts it into a language that the computer can understand.

Grammar

Every human language has its own set of rules, or Grammar, that specifies what is "correct" and what is "incorrect". Programming languages have Grammar too.

In any human languge, when people make grammatical mistakes the meaning can often still be understood by other humans. Computers however, aren't smart enough to do this. Becasue computers interpret instructions literally, even the smallest error will cause big probelms. This is means the code must adhere exactly to the grammar of the programming languge in order for the code to run.

Python Expressions

A python "expression" is a vaild python statement; it adheres to the rules of the python grammar. For example: print 1 + 1 is a valid python expression, but print 1 + (without a number at the end) is not valid.

Lesson 2: Variables & Strings

Python Variables

Variables in Python allow us to use names to refer to and keep track of values. To create a variable, use an assignment statement in the form name = expression. After the assignment statement, the name refers to the value of that expression.

For example, if the variable apples refers to the value 2, and the variable oranges refers to the variable 3, then the following code would print out 5: print apples + oranges

Assignment Statements

The equals sign in Python assignment statements does not mean "equal to". It means "assignment". The value on the right side of the = sign is assigned to the name on the left side of the sign.

The value of a variable does not have to be permanent. It can be changed by creating a second assignment statement where a new value is assigned to the already existing variable. When that variable is used again, it will refer to the new value.

Why Are Variables Useful?

Variables are uselful in many ways:

  • They allow programmers to store the value of data in names that make sense to humans, thereby improving code readability.
  • They give us a way of changing the value of something, like in the line: days = days - 1. This video does a good job of explaining the way values can change.

Strings

Strings are a sequence of characters surrounded by quotes. To allow for quotes to be used inside of strings, Python accepts either single, double or triple quotes around a string.

Just like numbers, strings can be printed and can also have variables assigned to them. The plus + operator can be used with two strings (but never a string and a number), to create a concatenation of the two strings. The multiplication * operator however can be used with a number and a string. print '!' * 3 would print out: !!!

Indexing Strings

Each character in a string has a position within that string. To index a part of a string, we use the following form: string[expression]

To index the string "Hello" using positive numbers, the H is 0, the e is 1 and so on. Therefore, print "Hello"[0] would print out H. To index "Hello" using negative numbers, start from the end. The o is -1, l is -2, l is -3, e is -4 and so on. Therefore, print "Hello"[-1] would print out o.

Selecting Sub-Sequences

To select a sub-sequence of a string, we use the operator :. The output is a string that is a sub-sequence of the characters in the search string starting from position start and ending with position stop, but not including that character.

Finding Strings in Strings

Find is a method. It is a built in procedure provided by Python where the output is the position in the search string where the first occurance of the target string is found. If the target string doesn't appear in the search string, the output will be -1.

Lesson 3: Input -> Function -> Output

What Is A Function?

A function, aka procedure, takes data as input, does somethings to that data (as defined by the function), and then produces an output. Python has built-in functions, like the .find function but it also allows the user to create functions.

Functions help programmers avoid repitition. Functions are tools that once created, can be reused forever. In other words, once a function has been defined, it never has to be defined again.

Making & Using Functions

Functions are made by starting with the line of code def and then giving a function name followed by the function's parameters in parantheses, followed by a semi-colon. These parameters will be replaced with values once the functions is used (called).

After the semi-colon is the "body" of the function. The body is what contains the code that specifies what to do with the input parameters. For example, the following code could be a definition of a function called sum:

def sum(a,b):
answer = a + b
return answer

To use a function, write the name of the function followed by the value(s) you want to give it in parentheses.

print sum(3,4)
>>>7

Keywords def & return

The keyword def is used to define a function. It tells python that what comes after it is a function. The keyword return tells python exactly what the function should return, or produce, as output. If the function doesn't have a return statement, you'll see behaviour like this:

def add_two(x):
answer = x + 2

new_number = add_two(7)
print new_number
>>>None

Lesson 4: Decisions & Repitition: If & While

Making Decisions

Decisions allow us to make code behave differently depending on the different values that we input. To make a decision, we first have to make a comparison so that we have a way to test and decide what to do.

There are many operators that are used for comparison. Examples include:

  • Equlality comparison: ==
  • Not equal to: !=
  • Greater than: >
  • Less than: <
  • Greater than or equal to: >=
  • Less than of equal to: <=
The output of a comparison is a Boolean value. It is either true, or false.

If Statements

If statements help us make decisions by making the code behave differently depending on the result of the comparison. They are written in the following form:

if < test expression >:
< block >


The < block > can be seen as the "body" of the if statement. It contains the code that will run if the test expression evaluates to true. If the test expression evaluates to false, the code in the body of the if statement will not execute.

It is important to note the indentation of the < block > . It shows us that it is inside of the if statement, and also where it ends. If the next statement is not indented, it will execute regardless of what the test expression evaluates to becasue it is not inside of the if statement's < block > .

If & Else

If we want to add another block of code to execute in the condition of the if comparison being false, then we can do this by adding the keyword else, followed by another < block > containing that code.

If the test expression following the if statement evaluates to true, the code in the follwoing indented < block > will execute. If the test expression evaluates to false, the code in the indented < block > following else will execute.

Or operator

The "or" operator follows this logic:

T and T = T
T and F = T
F and T = T
F and F = F

Therefore, if the first expression evaluates to true, the output value is true and the second expresion is not evaluated. If the first expression evaluates to false, the output value is the value of the second expression.

Examples

A function where the output is the biggest of the three numerical inputs:

def biggest(a,b,c):
if a > b:
if a > c:
return a
else:
return c
else:
if b > c:
return b
else:
return c

This problem can be solved much easier if we use the previoulsy defined function bigger(a,b):

def bigger(a,b):
if a > b:
return a
else:
return b

That solution looks like this:

def biggest(a,b,c):
return bigger(bigger(a,b),c)

Here, two calls to the function bigger are used to produce the output of the function biggest. The word bigger following the keyword return represents the second call to the operation bigger, while the word bigger inside of the parentheses represents the first call where the bigger of variable a and b is determined. The second call then compares the output of the first call with the last variable c to determine the biggest of those two.

Loops

A loop is a block of code, preceded by the keyword while, that repeats as long as the test expression following while evaluates to true. If the test expression evaluates to false, python moves on to the next instruction. In other words, as long as the test expression evaluates to true, the code in the body block of while will execute over and over again. If it evalutes to false, the code in the body is not executed and we move on to the next instruction. Here is an example of a loop structure:

i = 0
while i < 10:
print i
i = i + 1

The result will print out the numbers from 0-9. Once we print 9, the value of i changes to 10 and then when we evaluate the test expression, it evaluates to false so the loop stops.

Break Statements

Break statements provide a way to stop a loop even while the test expression is true. That structure looks like this:

while < test expression >:
< block>
if < break test >:
break
< block >

If the test expression evaluates to true, python evaluates the code inside the block. If the break test is false, python contines with the code in the block. Then python goes back to the test expression, if it evaluates to true it runs the code in the block. Python will repeat this cycle until the output of the break test is false. If this happens, instead of running the rest of the code in the block, python jumps out of the loop and continues with the next instruction.

Lesson 5: Structured Data: Lists & For Loops

Structured Data

Structured data is data that can be broken down into its individual elements and where we can perform operations on those elements. Examples include:

Strings, a sequence of characters. We can operate on sub-sequences of strings. For example: If we assign the variable s to the string ‘yabba!’, we can select an element of this string by indexing a certain position in the string using square brackets. Because we can access specific characters in a string, a string is an example of structured data.

Lists, a sequence of any element, allow us to reference any element(s) in the list and perform operations on it/them.

Lists

Lists are a sequence of any elements. There are no restrictions on the types of elements that can be put in a list. They can also contain more than one type of element. Here is an example of a list, which contains multiple types of elements. This list contains strings, numbers and even other lists.

mixed_up = [‘apple’, 3, ‘oranges’, 27, [1, 2, [‘alpha’, ‘beta’ ]]]

Lists allow us to nest lists inside of other lists. The last element in the list mixed_up is a list, containing another list. For lists that don’t fit on one line: create a new line after a comma, and indent it to let Python know that the elements on the second line are still part of the list.

We can access elements in a list the same way we index strings. If a list contains other lists, like the example below, we can access these nested elements by using the square brackets twice.

beatles = [[‘John’, 1940], [‘Paul’, 1942],
[‘George’, 1943], [‘Ringo’, 1940]]

print beatles[3]

>>> [‘Ringo’, 1940]

print beatles[3][0]

>>> Ringo
Mutation

Mutation means we can change the value of a list after we have created it. While lists do support mutation, it is extremely important to remember that strings do not. When we assign a variable to a string, we can change the value of the variable by making a second assignment statement and assigning that existing variable to a new string. It is important to note that the value of the variable is changing, not the value of the string itself.

When we mutate a list, we are changing the value of the list.

p = [‘H’, ‘e’, ‘l’, ‘l’, ‘o’]
p[o] = ‘Y’
print p
>>> [‘Y’, ‘e’, ‘l’, ‘l’, ‘o’]

In the first statement we assigned the variable p to a list containing some single character strings. The second assignment statement finds the position indicated on the left side p[0], and replaces it with the value on the right side, ‘Y’. Note, we did not create a new list, we changed the value of the existing list.

Aliasing

The term aliasing is used when there are two different ways to refer to the same object. What happens when we add the following assignment statements to the example above?

q = p
q[4] = '!'

print q
>>> ['Y', 'e', 'l', 'l', '!']
print p
>>> ['Y', 'e', 'l', 'l', '!']

The first statement introduces another variable, q, which also refers to the same list object that the variable p refers to. This is an example of aliasing; we have two variable names that refer to the same object. The second assignment statement shows why mutability is both powerful and tricky. Because two variables, p and q, now refer to the same object, mutating an element in variable q also changes the value of p. It is important to keep track of which variables refer to which object so that we don’t run into any problems when we make changes to any of those variables.

List Operations

Append is a built in method of Python that will add a new element to the end of a list. Append mutates the list. For example:

stooges = [‘Moe’, ‘Larry’, ‘Curly’]
stooges.append(‘Shemp’)
print stooges
>>> [‘Moe’, ‘Larry’, ‘Curly’, ‘Shemp’]

The + Operator behaves very similarly to the concatenation operation for strings. The + operator doesn’t mutate the list, it creates a new list. For example:

[0,1] + [2,3]
>>> [0,1,2,3]

The len operator (short for length) outputs the number of elements in the input. The len operator only counts the outer elements of a list. If one of the elements in the list being counted is a list, len will not count the elements within this list, only the list itself is counted as an element in the outer list. For example:

len( [‘a’, [‘b’, [‘c’]]] )
>>> 2
Loops on Lists

If we want to loop through the elements in a list, we can do so by using either a while loop or a for loop. The for loop provides a simpler way (less code to write) of doing this than the while loop. For loops take the following structure:

for < name > in < list >:
< block >

A for loop loops through a list in order and each time it assigns the name to one element in the list and evaluates the block. It does this until each element in the list has been looped through. A for loop makes defining the following function very easy.

my_list = [1, 2, [3, 4]]

def print_all_elements(p):
For e in p:
Print e

print_all_elements(my_list)
>>>
1
2
[3, 4]
More on List Operators in Loops
< list >.index( < value > )

Index is a method in Python that allows us to easily find where a particular value in a list first appears. If the value is not found, it produces an error.

< value > in < list >
< value > not in < list >

The in operator returns a Boolean value as its output. If the value is in the list, the output it True, otherwise output is false. The in operator can be preceded by the operator not, giving the reverse meaning.
